Proyecto Fine-Tuning LLM

Documentación del proceso de reentrenamiento de Gemma-2 270m con LoRA

Módulo: ASIR | Entorno: Mac/Linux (zsh)

1 Preparación del Entorno Virtual Setup

Creación del entorno virtual de Python (venv) e instalación de las librerías necesarias: mlx-lm, datasets, huggingface_hub y accelerate.

2 Autenticación en Hugging Face Auth

Inicio de sesión en el hub de Hugging Face para poder descargar los modelos base. Se utiliza el token de acceso personal.

3 Selección del Modelo Base Investigación

Elección del modelo gemma-3-270m-it (versión instruct) optimizado para MLX, ideal para pruebas rápidas de fine-tuning.

4 Creación del Dataset Inicial Data

Generación manual de los archivos train.jsonl y valid.jsonl con formato conversacional (role: user/assistant) específico para ASIR.

5 Error de Carga: IndexError Error

Primer intento fallido. El script arroja un IndexError: list index out of range, indicando que el dataset no se estaba leyendo correctamente o estaba vacío en ese momento.

6 Error de Batch Size Error

Al corregir la carga, surge un ValueError. El modelo requiere un batch_size=2 mínimo, pero el dataset de prueba solo tenía 1 ejemplo.

7 Solución: Ampliación del Dataset Fix

Se añaden más ejemplos al archivo train.jsonl mediante comandos echo para cumplir con los requisitos mínimos del batch size.

8 Entrenamiento Exitoso Success

Ejecución correcta del script mlx_lm.lora. Se completan las 50 iteraciones, la pérdida (loss) baja a 3.600 y se guardan los pesos (adapters) en my-gemma-lora.
